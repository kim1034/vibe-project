# 프로젝트 에이전트 규칙 (AGENTS.md)

이 문서는 AI·에이전트가 이 저장소에서 코드를 작성·수정할 때 **반드시 따를 규칙**을 정의합니다. 매 세션마다 같은 설명을 반복하지 않도록, 아래를 기본 전제로 삼습니다.

---

## 기술 스택 (고정)

| 영역 | 선택 |
|------|------|
| 프레임워크 | **Next.js** — **App Router** (`app/` 디렉터리)만 사용 |
| 언어 | **TypeScript** (`.ts`, `.tsx`) |
| 스타일 | **Tailwind CSS** |
| 데이터베이스·백엔드 | **Supabase** (클라이언트·서버 클라이언트·RLS·마이그레이션 등 프로젝트에 맞게 사용) |

---

## 반드시 지켜야 할 것 (DO)

- **App Router**로 라우팅·레이아웃·로딩·에러 경계를 구성한다. 새 페이지는 `app/` 아래에 `page.tsx`, `layout.tsx`, `loading.tsx`, `error.tsx` 등 App Router 관례를 따른다.
- **TypeScript**로만 작성한다. `any` 남발 대신 타입을 정의하거나 `unknown`과 타입 가드를 사용한다.
- **Tailwind** 유틸리티 클래스로 스타일을 맞춘다. 컴포넌트별 일관된 간격·타이포·색상 토큰을 유지한다.
- **Supabase**와 통신할 때는 공식 클라이언트 패턴을 따르고, 민감한 키는 서버 전용(`server-only` 등)으로 두며 클라이언트에 노출하지 않는다.
- 서버/클라이언트 경계를 명확히 한다. 필요 시 `"use client"`를 최소 범위에만 둔다.
- 접근성·시맨틱 HTML·반응형을 기본으로 고려한다.
- 프로젝트에 이미 있는 폴더 구조·네이밍·import 스타일을 우선 따른다.

---

## 하면 안 되는 것 (DON'T)

- **Pages Router**만을 전제로 한 코드(`pages/` 중심 라우팅, `getServerSideProps` 등 App Router와 혼용하지 않는 구식 단일 패턴)를 기본으로 제안하지 않는다. (레거시 유지가 명시된 경우만 예외.)
- **JavaScript만** 쓰도록 새 파일을 만들지 않는다. (기존 `.js` 마이그레이션 요청이 있을 때만 예외.)
- **인라인 스타일·별도 CSS 프레임워크**(예: styled-components, CSS Modules를 Tailwind와 무작정 병행)를 도입해 스타일 체계를 쪼개지 않는다. (프로젝트에 이미 합의된 경우만 예외.)
- **Supabase 서비스 키·비밀**을 클라이언트 번들·공개 저장소·로그에 넣지 않는다.
- **데이터베이스 스키마를 추측**만으로 바꾸지 않는다. 마이그레이션·RLS 정책은 사용자와 합의된 방식으로만 다룬다.
- 불필요한 **대규모 리팩터**나 요청 범위 밖 파일 수정으로 PR/디프를 키우지 않는다.

---

## 한 줄 요약

> Next.js **App Router** + **TypeScript** + **Tailwind** + **Supabase** — 이 네 가지를 깨지 않고, 위 DO/DON'T를 구분해 적용한다.
